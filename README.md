# Flashcard Image Carousel 🎴

โปรเจกต์นี้คือคอมโพเนนต์ Card ที่ซ้อนกันอยู่และสามารถคลิกเพื่อ "สลับการ์ดใบหน้าสุดไปไว้หลังสุด" โดยมี Animation การสไลด์ออกทางซ้ายและมุดกลับเข้าไปซ้อนด้านหลัง เหมือนกับการสับการ์ดคำศัพท์ (Flashcard)

เอกสารนี้จะอธิบาย **"ขั้นตอนและแนวคิดการทำงาน"** อย่างละเอียด เพื่อให้นักเรียนสามารถทำความเข้าใจและนำเทคนิคนี้ไปประยุกต์ใช้กับงานอื่นๆ ได้

---

## 💡 แนวคิดหลัก (Core Concept)

เพื่อให้การสลับการ์ดดูเป็นธรรมชาติ เราจะแบ่ง Animation ออกเป็น 2 จังหวะ:
1. **โยกออก (Animate Out):** การ์ดใบหน้าสุดถูกเลื่อนออกไปทางซ้าย (ยังอยู่เลเยอร์บนสุด)
2. **มุดกลับ (Slide Back):** การ์ดใบนั้นถูกลดเลเยอร์ลงไปอยู่ล่างสุด แล้วเลื่อนกลับมาที่ตำแหน่งเดิมตรงกลาง ทำให้ดูเหมือนมันสอดเข้าไปอยู่ใต้กองการ์ด

### ⚠️ ปัญหาที่มักพบเมื่อทำ Animation กับ React
หลายคนอาจจะใช้วิธี **"นำข้อมูลตำแหน่งแรกของ Array ไปต่อท้าย Array"** เพื่อสลับการ์ด
แต่การทำแบบนั้นจะทำให้ React **สลับตำแหน่งของ Element ใน DOM (DOM Reordering)** 
เมื่อ Element ถูกย้ายที่ใน DOM เบราว์เซอร์จะ **ตัดจบ CSS Transition ทันที** ทำให้การ์ดที่กำลังโยกออก "หายวับ" และกระโดดไปอยู่ข้างหลังโดยไม่มีการสไลด์

### ✅ วิธีแก้ปัญหา (The Solution)
**"ห้ามสลับลำดับใน Array เด็ดขาด แต่ให้ใช้วิธีเลื่อน Class แทน"**
เราจะใช้ตัวแปรชื่อ `offset` เพื่อเป็นตัวบอกว่า ตอนนี้การ์ดใบไหนควรจะได้ Class สำหรับอยู่หน้าสุด โดยที่โครงสร้าง HTML (DOM) ยังคงเรียงลำดับเหมือนเดิมทุกประการ ทำให้ CSS Transition สามารถแสดงผลได้อย่างสมบูรณ์

## 🛠️ ขั้นตอนการทำงาน (Step-by-Step)

### ขั้นที่ 1: การเตรียม State ใน React (`App.tsx`)

เราต้องการ State 2 ตัวหลักๆ คือ:
```tsx
// เก็บว่าขยับคิวไปแล้วกี่ครั้ง (ใช้หมุนเวียนตำแหน่งการ์ด)
const [offset, setOffset] = useState(0);

// เก็บ ID ของการ์ดที่กำลังแสดง Animation สไลด์ออก
const [animatingId, setAnimatingId] = useState<number | null>(null);
```

### ขั้นที่ 2: การคำนวณตำแหน่งภาพ (Visual Position)
ในฟังก์ชัน `map` เพื่อสร้างการ์ด เราจะใช้สูตรคณิตศาสตร์แบบ Modulo (`%`) เพื่อคำนวณตำแหน่งที่จะแสดงผล:
```tsx
// สูตร: (ตำแหน่งดั้งเดิม + ค่าการเลื่อน) หารเอาเศษด้วยจำนวนการ์ด
const posIndex = (index + offset) % cards.length;
```
- ถ้า `posIndex === 0` หมายถึงการ์ดใบนี้อยู่ **หน้าสุด**
- ถ้า `posIndex === 1` หมายถึงการ์ดใบนี้อยู่ **ตรงกลาง**
- ถ้า `posIndex === 2` หมายถึงการ์ดใบนี้อยู่ **หลังสุด**

เรานำ `posIndex` ไปกำหนด Class เช่น `.card-pos-0`, `.card-pos-1` ให้กับการ์ด

### ขั้นที่ 3: การกำหนด CSS เพื่อจัดตำแหน่งและ Animation (`index.css`)
เราเตรียม Class สำหรับแต่ละตำแหน่ง โดยกำหนด `z-index` (ความสูง/การซ้อนทับ) และ `transform` (การหมุน/การเลื่อน)
```css
.card-inner {
  /* กำหนดให้ความเปลี่ยนแปลงทุกอย่างค่อยๆ เกิดขึ้น (Animation) ในเวลา 0.4 วินาที */
  transition: transform 0.4s ease-in-out;
}

.card-pos-0 { z-index: 3; transform: translateX(0) rotate(0deg); }
.card-pos-1 { z-index: 2; transform: rotate(8deg); }
.card-pos-2 { z-index: 1; transform: rotate(16deg); }
```

### ขั้นที่ 4: จังหวะคลิก (โยกออก)
เมื่อผู้ใช้คลิกการ์ดหน้าสุด ฟังก์ชัน `handleSwap` จะเริ่มทำงาน:
1. เก็บค่า ID ลงใน `animatingId`
2. การ์ดใบนั้นจะได้รับ Class พิเศษชื่อ `.animate-out` ทันที

```css
.animate-out {
  /* เลื่อนไปทางซ้าย -140%, เลื่อนลง 20px, และเอียง -15 องศา */
  transform: translateX(-140%) translateY(20px) rotate(-15deg) !important;
  /* รักษาให้อยู่บนสุดเสมอในขณะที่กำลังเลื่อนออก */
  z-index: 4 !important; 
}
```

### ขั้นที่ 5: จังหวะสลับตำแหน่ง (มุดกลับ)
ใน `handleSwap` จะมีคำสั่งหน่วงเวลา (`setTimeout`) ที่ตั้งเวลาไว้ **400 มิลลิวินาที** ซึ่งตรงกับเวลาของ CSS Transition (0.4s) พอดี:

```tsx
setTimeout(() => {
  // เปลี่ยนค่า offset ให้หมุนเวียน (ลดลง 1)
  setOffset((prev) => (prev - 1 + cards.length) % cards.length);
  // เอาคลาส .animate-out ออก
  setAnimatingId(null);
}, 400);
```

**เกิดอะไรขึ้นในเสี้ยววินาทีนี้?**
1. คลาส `.animate-out` ถูกถอดออก
2. ค่า `offset` เปลี่ยนไป ทำให้การ์ดใบนี้ถูกคำนวณ `posIndex` ใหม่กลายเป็น `2` (หลังสุด)
3. การ์ดได้รับคลาส `.card-pos-2` แทน
4. ค่า `z-index` ตกจาก 4 ลงมาเป็น 1 ทันที ทำให้ตัวการ์ดร่วงลงไปอยู่ "หลังสุด"
5. เบราว์เซอร์ตรวจพบว่า `transform` เปลี่ยนจากฝั่งซ้าย (`translateX(-140%)`) กลับมาเป็นปกติ (`rotate(16deg)`) 
6. เบราว์เซอร์จึงจัดการสร้าง Animation นำการ์ดสไลด์กลับมาที่กองอย่างนุ่มนวล **(แต่คราวนี้สไลด์จากด้านหลังของกองการ์ด!)**

## 🎯 บทสรุปสิ่งที่ได้เรียนรู้
- **CSS Transitions** จะทำงานได้สมบูรณ์ก็ต่อเมื่อ DOM Element นั้นไม่ได้ถูกทำลายหรือย้ายลำดับใน HTML
- **z-index** ไม่มีการทำ Animation ระหว่างช่วง มันจะเปลี่ยนทันที เราจึงสามารถใช้ประโยชน์นี้เพื่อ "สลับเลเยอร์แบบทันทีทันใด" ร่วมกับการทำ Animation การเคลื่อนที่ (`transform`) ได้
- **Modulo (`%`)** เป็นเครื่องมือทางคณิตศาสตร์ที่ทรงพลังมากในการทำวงล้อหมุนวน (Circular logic) ไม่ว่าจะเป็น Carousel, Slider หรือการหมุนเวียน Class
